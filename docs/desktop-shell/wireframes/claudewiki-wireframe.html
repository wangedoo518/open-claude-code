Transformer 和 RNN 的核心区别是什么？
根据 [[comparisons/rnn-vs-transformer]]：

核心区别：Transformer 用自注意力替代了 RNN 的循环结构，实现了并行计算。

RNN 按时间步顺序处理，长距离依赖容易衰减；Transformer 通过注意力矩阵直接建模任意位置关系。

来源: concepts/transformer-arch · people/vaswani

Transformer 架构

confidence: high 3 sources 更新于 2h 前

定义：基于自注意力机制的序列转换模型架构，由 Vaswani 等人在 2017 年提出，彻底改变了 NLP 领域的研究范式。

要点

  • Self-Attention 是核心创新 — 允许模型在处理每个位置时关注输入序列的所有其他位置
  • 位置编码解决序列顺序 — 由于不像 RNN 有内在的序列处理顺序，需要显式注入位置信息
  • 多头注意力捕获不同层次关系 — 让模型同时从多个表示子空间学习信息
  • Layer Normalization + Residual Connection — 保证深层网络的训练稳定性
  • Encoder-Decoder 结构可根据任务灵活组合 — GPT 只用 Decoder，BERT 只用 Encoder

关键引用

Attention Is All You Need — 这篇论文的标题本身就是最好的一句话总结。

代码示例

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.d_k = d_model // n_heads
        self.heads = n_heads
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
